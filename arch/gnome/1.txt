===Просмотр настроек ядра===
Проверка версии ос:
uname -a
Проверка параметров, которые передаются ядру при загрузке системы (Boot параметры):
cat /proc/cmdline
Просмотр параметров, связанных с управлением памяти(swap - выгрузка данных из оперативки на жесткий диск) и реакцией системы на критические ошибки;
параметр vm.swappiness от 0(избегание использования swap раздела) до 100(активное использование swap, даже когда в оперативной памяти нет места), настройка будет временной (до перезагрузки);
kernel.panic от 0(Система не перезагружается после критической ошибки) до n секунд, после которых система перезагружается; 
все остальные параметры kernel.panic показывают либо 0 (продолжить работу после ошибки), либо 1 – принудительно вызвать kernel panic (перезагрузка для безопасности), например: kernel.panic_on_io_nmi - ошибки ввода-вывода:
sysctl -a | grep -e "vm.swappiness" -e "kernel.panic"
Просмотр установленных модулей ядра:
lsmod

Дополнительно можно продемонстрировать настройку swappiness(настройка, указывающая системе, когда начинать 'выгружать' неиспользуемые данные из оперативной памяти на жесткий диск);
от 0(избегание использования swap раздела) до 100(активное использование swap, даже когда в оперативной памяти нет места), настройка будет временной (до перезагрузки):
sudo sysctl vm.swappiness=30


===SSH===
Настройки -> system -> Secure Shell -> нажимаем ✓
Узнаем ip: 
ip addr
С основного ПК прописываем:
ssh login@ip (Пример: user@10.201.83.146) 
Вводим для проверки ls 


===Удаленный доступ===
Настройки -> system -> Remote Desktop -> включаем две верхние ✓
Переходим в Remote Login, включаем его, далее в Login Details вводим данные логина и пароля
В терминале прописываем ip addr и смотрим ip 
Выключаем ВМ и правой кнопкой мыши по ВМ клонируем её, далее нажимаем клонировать (Сгенерировать новые mac адреса)
Запускаемся со второй ВМ и запускаем программу Connections
Нажимаем + и вводим ip ВМ, на которой включили удалённый доступ
Нажимаем Verify, затем вводим данные, которые добавляли ранее
После запуска вводим пароль от учётки, нажимаем force quite, а затем выключаем подключение 
Снова подключаемся, вводим пароль от учётки

===Виртуальный принтер===
Установка компонентов:
sudo pacman -Syu cups cups-pdf ghostscript
Запуск служб печати:
sudo systemctl enable cups.service
sudo systemctl start cups.service
Создание папки для pdf-файлов:
mkdir -p ~/название_папки
Открываем файл конфигурации CUPS:
sudo nano /etc/cups/cups-pdf.conf
Находим строку, начинающуюся с Out и меняем ее на:
Out /home/имя_пользователя/название_папки_из_3_пункта
Сохраняем и закрываем:
Ctrl + x -> y -> ENTER
Перезапуск CUPS:
sudo systemctl restart cups.service

Пишем в браузере ссылку:
http://localhost:631
Выбираем «Администрирование», затем «Принтеры» и «Добавить принтер», вводим логин и пароль
В списке доступных вариантов выбрать «CUPS-PDF (Virtual PDF Printer)»
Настроить параметры принтера: задать его имя, выбрать «разрешить совместный доступ к этому принтеру»
Выбрать «Generic»
Выбрать драйвер для принтера — «Generic CUPS-PDF Printer (w/options) (en)». Нажать «Добавить принтер»
Во вкладе принтеры. Настроить параметры печати по умолчанию.
Чтобы проверить работает ли принтер:

Выбираем «Печать пробной страницы»
Созданные файлы появятся по пути /home/имя_пользователя/название_папки_из_3_пункта
